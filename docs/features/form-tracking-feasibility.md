## 検証で確定した設計知見（本実装 46-form-core.js に引き継ぐ）

1. **距離・速度はすべて胴体長（肩中点〜腰中点）で正規化する。**
   画面正規化座標の生値でしきい値を切ると、カメラ距離（構図）で判定が
   破綻する。archery-master 版のしきい値（anchorDist<0.19 等）はこの理由で
   全身撮影時に甘くなる。胴体長比なら構図非依存。
2. **検出は生値、表示のみ EMA 平滑化。**
   EMA(α=0.38) をかけた系列でリリース検出すると速度スパイクが鈍り
   取りこぼす。平滑化は HUD/オーバーレイ表示専用にする。
3. **リリース判定は「短窓の離脱量」を主条件にする。**
   低フレームレート（15-20fps）ではリリース瞬間の速度は 1-2 フレームしか
   映らない。直近 250ms でのアンカー最接近点からの離脱量
   （胴体長比 >0.18）を主条件、速度スパイク（>1.2 胴体長/s）を補助 OR 条件に
   すると、取りこぼしがなくなり、かつレットダウン（漸進的変位）を
   誤検出しない。**窓を 450ms に広げるとレットダウンを誤検出する**
   （合成テストで確認済み）。
4. **フェーズはヒステリシス付きステートマシン。**
   FULL_DRAW = アンカー圏（アンカー比<0.35）に 350ms 以上滞在 + 引き手肘>125°。
   リリース後 1000ms は不応期。合成シーケンスで
   SETUP→DRAWING→ANCHORING→FULL_DRAW→RELEASE→FOLLOW の遷移、
   連続 2 射、レットダウン非発火を検証済み。
5. **エリート基準ガウス採点（archery-master 由来）はそのまま流用可。**
   弓手肘 172°±9 / 引き手肘 152°±14 / 押し引き力線 ほか 7 項目。
   合成フルドロー姿勢で妥当なスコアが出ることを確認。
6. 検出低下（visibility<=0.5）の部位名表示は、弓による隠れの切り分けに有効。

## 既定しきい値（プロトタイプで実射調整済みの初期値）

```
CLOSE_IN: 0.35        // アンカー圏に入るアンカー比（対胴体長）
FULLDRAW_MS: 350      // FULL_DRAW 判定に必要な滞在時間
RELEASE_RISE: 0.18    // 250ms 窓での離脱量しきい値（感度で除算）
RELEASE_TH: 1.2       // 速度補助条件（胴体長/秒）
RISE_WINDOW_MS: 250   // 離脱量の観測窓（伸ばすとレットダウン誤検出）
REFRACTORY: 1000      // リリース後の不応期
```

ユーザー調整は「感度」1 変数（しきい値を感度で除算）で足りる。
