# Archery Note デザイン言語 v2 — "Field Instrument"

UI の正本。以降の UI PR はすべてこの文書に従う。違反はレビューで差し戻す。
ここに無い判断が必要になったら実装せず設計者（ハブ）へ返す。

v2 の改訂理由: v1 は「ちゃんと作ってある」段階で止まり、洗練に達しなかった。
敗因は (1) 金を面で使い黄土色化した (2) pill 由来の丸さでぼってりした
(3) 黒パネルが渋滞し呼吸が浅い (4) 説明過多 (5) 主役過多。v2 はこの5点を禁則化する。

## 方向（一文）

**墨とインクの精密な計器。金は光の一筋だけ。**

## 0. 最上位原則 — 主役は1画面に1つ

各画面は主役を1つ宣言し、他の全要素は主役より**最低1段階静か**（小さい/細い/淡い）にする。

| 画面           | 主役       | 静かにするもの                                            |
| -------------- | ---------- | --------------------------------------------------------- |
| 記録（開始）   | 開始 CTA   | 条件選択・前回情報                                        |
| 記録（記録中） | **的**     | HUD・ドック・チップ（全て無地・小型・低コントラスト側へ） |
| 分析           | 今日の結論 | 全カード                                                  |
| 履歴           | 記録リスト | ヒーロー（1行に縮小）                                     |
| サイト         | 提案の数値 | 台帳・詳しく                                              |
| 用具           | 台帳       | ヒーロー                                                  |

同格の強さの要素が2つ並んだらどちらかを降格する。降格先は「小さく」ではなく
「淡く・細く・無地に」が第一選択（サイズを縮めると窮屈になる）。

## 1. 色 — 墨がブランド、金は糸

- **金の面（塗り）は全面禁止**。ボタン・チップ・バッジ・帯のいずれも金背景にしない
- **主 CTA・選択状態は「反転」**: light = 墨面 `#1b1b18`＋白文字、dark = 紙面＋墨文字。
  モノクロ反転が本アプリの「強い」の表現。金で強調しない
- **金は糸としてのみ**: 2px 以下の線（アクティブタブの下線・現在地ドット・hairline アクセント）、
  ライブ中の主役数値の文字色、の2用途に限定。値は light `#b08a2e` / dark `#d9b25a`
  （小面積前提でわずかに明るく。面で使わないので黄土化しない）
- ベースの生成りを弱める: light 背景は `#faf9f6` 系のクリーンなオフホワイトへ
  （黄味の強い紙色は「古さ」の一因）。dark は `#101110` 系の純に近い墨
- 的色（金/赤/青/黒）はデータ専用。従来どおり

## 2. 形 — pill 全廃、直線のキレ

- 角丸: カード/パネル **8px**、ボタン **8px**、入力/チップ **6px**。**pill（999px）は全廃**
  （チップも角丸 6px の矩形。丸さで柔らかくせず、余白で優しくする）
- タブの現在地は「塗り」でなく**金の下線（2px）＋ラベル墨強調**
- カードは原則**無地**（背景=地、境界は hairline 1本か余白のみ）。
  「面で区切る」のはモーダルと記録中ドックだけ
- カード内カード禁止。ネストが要るなら罫線と字下げで表現

## 3. タイポグラフィ — 大きいのは数字だけ

- **画面内で「大きく太い」ことが許されるのは主役の数値（と記録中の的）だけ**
- 見出しは全てラベル（11–12px・字間 0.08em・淡色）。v1 の「大見出し＋説明文」は禁止
- **説明文ダイエット**: UI に機能説明の文章を置かない。
  1行を超える説明は削除するか details へ。「〜できます」「〜が表示されます」形の文は原則削除
- 一言解釈（分析）は例外として1行残す。ただし 15–40字、句点1つまで
- 数値は tabular-nums 維持

## 4. 余白 — 8pt リズムと呼吸

- 縦リズムは 8 の倍数のみ（セクション間 32、カード内 12/16）
- **固定バー（タブ・ドック・HUD）は内側に十分な padding**を持ち、隣接要素と最低 16px 空ける
- 記録中画面の強い面は**ドック1つまで**: HUD はパネルを持たない
  （地の上に数値が置いてあるだけ。枠・面・影なし）。黒い帯を縦に並べない
- 詰まりの点検基準: スクリーンショットを縮小して見たとき、黒い塊が2つ以上
  接していたら失敗

## 5. アイコン — 計器の刻印

- 24px グリッド・stroke 1.5px・**butt cap（角端）**・パス3本以内・装飾なし
- タブ5種は専用に再設計する（自作感の排除が目的。幾何学に還元する）:
  記録=レティクル（円＋中心点＋上下左右の短いティック）、履歴=台帳行（左に日付ティックの
  ある3本線）、分析=単一の折れ線＋終端ドット、サイト=縦スケール＋ポインタ、
  用具=リムの単曲線＋弦（1曲線1直線）
- 実装前に**アイコンシート（全アイコンを1枚に並べた画像）を作り監督レビューを通す**こと
- サイズ違いで線の太さが変わらないよう viewBox 統一

## 6. モーション（v1 から変更なし）

4目的制（因果/方向/状態/注意）・CSS コメント宣言必須・トークンの時間とイージングのみ・
reduced-motion 対応。新しい動きの追加は主役を強めるものに限る。

## 7. プロ品質チェックリスト（改訂）

- [ ] 金の「塗り」が 1px も無い（線・ドット・文字色のみ）
- [ ] pill が無い（border-radius は 6/8px と 0 のみ）
- [ ] 主役が1つで、同格の強さの要素が並んでいない
- [ ] 機能説明の文章が無い（削除 or details）
- [ ] 見出しが全てラベル様式（大見出しなし）
- [ ] 縮小視で黒い塊が接していない
- [ ] トークン外の値なし / tabular-nums / 全状態定義 / dark 検証 / 3ビューポート（v1 同様）

## 8. 工程の掟（v1 から変更なし）

1 PR = 1画面または基盤1件。ロジック・storage・SW 不変。check-ui/check-app/e2e を
同一 PR で同期。data-testid 移行を継続。
