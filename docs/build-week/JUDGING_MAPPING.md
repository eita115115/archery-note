# Judging Mapping

The official [OpenAI Build Week page](https://openai.com/build-week/) and [Devpost challenge rules](https://openai.devpost.com/rules) were checked on 2026-07-15. Stage Two uses four equally weighted criteria.

| Official criterion           | Archery Note evidence                                                                                                                                                                                                       |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Technological Implementation | A working, non-trivial local-first PWA; GPT-5.6/Codex-assisted architecture review and implementation; pure analysis functions; 38 security regressions; 41 Playwright tests; dated Build Week commits and development log. |
| Design                       | A coherent phone-first record-to-analysis flow; restrained dashboard inside the existing Analysis tab; 7/30/90-day controls; dark mode; fictional removable demo data; visible confidence and evidence.                     |
| Potential Impact             | University club archers and beginners can turn disconnected private practice records into a concrete next action even at ranges with unreliable connectivity.                                                               |
| Quality of the Idea          | The app combines scoring, grouping, form, equipment and sight history into explainable self-comparison rather than cloud inference or an opaque coaching score.                                                             |

## Official submission facts

- Recommended category: **Apps for Your Life**
- Submission deadline: July 21, 2026 at 5:00 PM Pacific Time
- Existing projects are judged only on meaningful work added during the submission period.
- The demo must be a public YouTube video under three minutes with audio explaining the project and the use of Codex and GPT-5.6.
- The submission must include a repository URL, relevant license, README instructions, and the `/feedback` Codex Session ID for the main project thread.
