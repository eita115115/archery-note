# Archery Note 🏹

アーチェリーの得点記録＆サイト調整サポートアプリ（スマホ対応・単一HTML）。

## 使い方

ローカルで開くだけでも動きます（`index.html` をブラウザで開く）。
スマホで使うには、どこかに置いて（おすすめは GitHub Pages）URLを開き、
ブラウザのメニューから **「ホーム画面に追加」** するとアプリのように使えます。
HTTPSで開けばオフラインでも動作します（Service Worker対応）。

ローカルでの動作確認用サーバー:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File serve.ps1
# → http://localhost:8741/
```

## 機能

- **記録**: 的面をタップ＆ドラッグで着弾位置を入力（押したまま0.4秒で微調整モード、拡大レンズ付き）。点数は座標から自動採点（X/10〜1/M）。エンド制で集計。距離は70/50/30/18m＋自由設定、的は122/80/60/40cm。競技ラウンドと風向/風速メモも任意で記録できます。
- **体系UI**: 記録画面に「データで育つ記録アプリ」の短い説明と「練習記録 / 校正用 / 状態確認」の入口を配置。READMEを読まなくても、点取りだけで始めて、サイト値・風・用具を残すほど提案が具体化する流れが分かる構成です。サイト画面は提案を先に見せ、モデル診断・校正状況・分析根拠・クリック換算は必要な時だけ開けます。
- **履歴**: 練習一覧・着弾プロット・エンド表・グルーピング中心/半径(RMS)・過去中心の分布/偏移・距離別平均点の推移グラフ。用具・距離・ラウンドで絞り込み、条件別の変化を確認できます。
- **サイト台帳**: セッティング×距離ごとにサイト値（上下/左右）を日付つきで台帳管理。練習終了時にワンタップで台帳へ記録。
- **校正モード**: 18/30/50/70m の実測サイト値をまとめて入力し、個人補正データと距離別サイト予測を育てられます。
- **調整アドバイス**:
  - 明らかな外れ値を除いたグルーピング中心から、信頼度・的サイズ・距離・ポンド・矢重量・初速・弾道シミュレーション・距離別サイト値を加味して「サイトをどちらへ・何mm（・何クリック）」を提案
  - 楕円ロバスト統計で、縦長・横長・斜め方向のグルーピングと外れ値をより精密に判定
  - 「動かす / 少量調整 / 保留 / 射形優先 / 風を考慮」の判断ラベルを表示し、無理にサイトを動かさない場面も明確化
  - 風・グルーピング形状・用具入力の不足から、次に確認すべき練習テーマを提案
  - 同じ用具・距離・的サイズの過去データと今回の中心を比較する個人モデルで、「過去と一致するズレ」か「今回だけの可能性」かを判定
  - 練習ごとの判断信頼度（高/中/低）と次アクションを表示し、履歴一覧にも信頼度バッジを表示
  - サイト値を数値で記録した複数回の練習から、練習信頼度・風・本数を加味した外れ値に強い回帰分析で「ズレが0になるサイト値」を推定
  - mm目安はアイ〜サイト距離（設定可、既定850mm）と、空気密度・抗力係数・矢径・ベイン・実測初速を加味した3D弾道エンジンから計算。RK4積分で向かい風/追い風/左右風/巻き風の風ベクトルと横流れも推定
  - 横風で説明できる左右ズレは、サイトを動かし過ぎないように補正量を控えめにします。クリック換算は「1クリック=◯cm@70m」を一度測って登録すると有効（他距離へ自動換算）
  - 履歴内のサイト値つき練習・横風メモ・距離別サイト台帳から、風の効きや上下/左右の目盛り効きを個人校正。明示クリック校正がない場合も履歴推定の目盛り換算を表示
  - 実測サイトマークが増えると、個人補正データの充実度と個人データ準備度を表示し、距離別サイト予測は必要に応じてカーブ近似へ切り替え
- **用具**: 弓・リム・サイト・レスト・弦・ポンド・引き尺・矢重量・実測初速・ブレースハイト等のセッティング管理と変更履歴の自動記録。候補式プルダウンで主要用具を選びやすくし、カタログ掲載シャフト名からスパイン・直径・GPI・総矢重量を推定入力。初回セットアップウィザード、直近練習との用具比較、矢セット重量差・真直度・FOC・ベイン・気温/標高/湿度などの個体差/環境項目に対応。
- **用具補助**: 実測ポンド・矢尺・ポイント重量から、カーボンシャフトのスパイン初期候補を表示。現在入力している番手が硬め/柔らかめ寄りの可能性も確認できます。スタビライザー/ウエイト、タブ/リリーサーもセッティング履歴に記録できます。
- **データ**: 端末内（localStorage）保存。JSONでバックアップ/復元可能。保存操作に合わせて端末内へ自動退避スナップショットを保持し、設定から復元可能。誤削除した練習・サイト値・用具は設定のゴミ箱から復元できます。
- **書き出し**: 練習履歴のCSV出力と、各練習のスコアカードSVG画像保存に対応。CSVには判断信頼度と個人モデル状態も含めます。
- **分析強化**: 軸別ばらつき（上下/左右の±1σ）と縦長・横長グルーピングの原因ヒント。外れ値の自動除外。
- **サイトマーク予測**: 2距離以上の実測サイト値から他距離の上下サイト値を直線近似で予測。
- **編集**: 記録中のエンドの開き直し（✏）、履歴からの過去セッション編集（上書き保存）。
- **三つ目的**: 40cm縦3スポット（10〜6点、6点未満はM）。分析時は1スポットに重ねて表示。
- **得点分布・月間サマリー**: 履歴タブにヒストグラムと月別の練習量・平均。
- **更新通知**: 新バージョン公開時にバナー表示（タップで更新）。
- **ダークモード**: ⚙設定から自動/ライト/ダークを選択（既定は端末に追従）。
- **線かみ判定**: 矢の中心点からアプリ上の矢円半径を引いて採点（表示された矢円が線に少しでも触れていれば内側の点数）。微調整モード中は線かみ時にカーソルが緑、線なし時に赤で表示されます。
- **環境対応**: 連続入力中の誤ズームを抑制し、PointerEvent非対応環境ではタッチ/マウス入力へフォールバック。オフライン時のトップページ復帰も補強。

## 更新時の注意

公開のたびに `index.html` 内の `APP_VER` と `version.json` の `v` を**同じ値に揃えて+1**すること（更新通知バナーの仕組み）。

更新前の検証:

```powershell
& 'C:\Users\eita2\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' tools\check-app.js
& 'C:\Users\eita2\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' tools\check-ui.js
git diff --check
```

`tools\check-ui.js` はローカルの Chrome/Edge を使って、スマホ幅・小型スマホ幅・PC幅のスクリーンショットを `artifacts\ui-smoke\` に生成します。生成物は `.gitignore` で除外されます。

カタログPDFから用具候補を下ごしらえする開発用ツール:

```powershell
.\.venv\Scripts\python.exe tools\extract-catalog.py artifacts\catalog\SBA2026_27.pdf
```

抽出結果とPDF本体は `artifacts\catalog\` に置き、Gitには含めません。アプリへ入れる候補は、人の目でノイズを落としてから `index.html` に反映します。

このCodex環境から `.git` へ書き込めない場合は、最後のコミットだけPowerShellで実行してください。

## ファイル

- `index.html` — アプリ本体（依存ライブラリなし）
- `manifest.json` / `sw.js` / `icon.svg` — PWA用（ホーム画面追加・オフライン動作）
- `serve.ps1` — ローカル確認用の簡易サーバー
- `tools/check-app.js` — 構文・公開番号・代表演算の検証スクリプト
- `tools/check-ui.js` — Chrome/Edgeを使ったスマホ幅・PC幅のUIスモーク検査
- `tools/extract-catalog.py` — カタログPDFから用具名候補を抽出する開発補助ツール
